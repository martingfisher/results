## How to use in Webflow
1. Build each `.page` as one Webflow page template section-by-section.
2. Treat each `.block` label as content type, not final styling.
3. Suggested section class naming:
- `sec-hero`
- `sec-risks`
- `sec-role-routing`
- `sec-services`
- `sec-process`
- `sec-proof`
- `sec-final-cta`
4. Suggested component naming:
- `cmp-nav`
- `cmp-cta-primary`
- `cmp-cta-secondary`
- `cmp-card-role`
- `cmp-card-risk`
- `cmp-step`
- `cmp-case-card`
- `cmp-faq-row`

## How to use in Figma
1. Create one page per website template.
2. Use a 12-column desktop grid and 4-column mobile grid.
3. Convert repeating items to components:
- CTA pair
- role card
- risk card
- process step row
- case card
- final CTA band
4. Keep these as low-fidelity grayscale blocks for stakeholder sign-off first.

## Motion intent (for later high-fidelity)
- Hero intro: text stagger + ambient abstract panel movement.
- Card grids: fade-up stagger on enter viewport.
- Process section: scroll-linked progress indicator.
- Header: sticky after first fold.
- Contact form: progressive disclosure for optional fields.
